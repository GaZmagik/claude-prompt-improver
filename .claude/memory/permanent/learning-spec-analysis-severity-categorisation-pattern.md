Group spec-implementation misalignments into HIGH (risk/ambiguity), MEDIUM (clarity), LOW (consistency). Fixes HIGH first prevents implementation rework, MEDIUM enables correct implementation, LOW is polish. Applied to 13 issues in prompt-improver spec.
